# Envinromnet
* Ubuntu 22.04
* Intel Core i5 Gen9

# Requirements
* avr-gcc
* avr-gdb
* avr-libc
* avr-binutils
    * avr-objcopy
    * avr-objdump
* qemu-system-avr

# Installation
## Arch Linux
```bash
$ sudo pacman -Sy avr-binutils avr-gcc avr-gdb qemu-system-avr
```

## Ubuntu / Debian
```bash
$ sudo apt install -y avr-binutils avr-gcc avr-gdb qemu-system-avr
```

## main.cのコンパイル
まずmain.cを作成する。

```c
#include <avr/io.h>
int main(void) {
    DDRD = 0xFF;
    TCCR0B = (1 << CS00);
    while(1) {
        PORTD = PIND + 1;
    }
}
```

保存したら`avr-gcc`でコンパイルする。
この際、適用するチップを`-mmcu`オプションで指定する。

```bash
$ avr-gcc -Wall -g -mmcu=atmega2560 -o sample.elf main.c
```

`-g`と`-Wall`はデバッグ情報の付加と、警告出力のスイッチ。

`-mmcu`で指定できるデバイスは`man avr-gcc`で確認できる。manの中で`/-mmcu`で検索するとAVRの一覧が見つかる。
参考のため以下に転載する。指定の仕方は`-mmcu=atmega2560`のようにする。

## QEMUでのエミュレーション
コンパイルして出力された`sample.elf`を`-bios`に指定する。
```bash
$ qemu-system-avr -gdb tcp::1234 -S -M mega2560 -bios sample.elf 
```

別のターミナルを開き`sample.elf`のディレクトリで以下のコマンドを実行する。

```bash
$ avr-gdb sample.elf

```

gdbに入ったら`target remote localhost:1234`を実行する。
